PRE-FINAL EXAM
A Laboratory Exam for Web Services that’s practical, hands-on, and aligned with what students should
demonstrate in a lab setting. This exam will test their ability to integrate, deploy, and troubleshoot web
services using APIs and hosting fundamentals.

🧪 Laboratory Exam: Web Services
Instructions
Duration: Be able to present your output
Tools allowed: Django/ Laravel/ Python/Flask, internet access for API documentation, free hosting
platforms (Heroku, Firebase, or PythonAnywhere).
Deliverables: Working code + short documentation (README).

Part A: API Integration (40 points)
Task:
1. Create a Flask(Laravel/Django) app with a form that accepts a city name.
2. Use the OpenWeatherMap API to fetch and display:
Current temperature
Weather condition
Country code of the city
Bonus (5 points): Add error handling for invalid city names.

Part B: Multi-Service Dashboard (40 points)
Task:
1. Extend the app to include a second widget:
Use the NewsAPI to fetch the top 3 headlines related to “Technology.”
Display them alongside the weather data.
Bonus (5 points): Allow the user to enter a topic (e.g., “Sports”) and fetch headlines dynamically.

Part C: Web Hosting Deployment (20 points)
Task:
1. Deploy your app on a free hosting platform (Heroku, PythonAnywhere, or Firebase Hosting).
2. Submit the live URL and a screenshot of the deployed app.

(Using Flask)
Part A: API Integration (Weather)
Expected Steps:
1. Import Flask and requests.
2. Create a route /weather that accepts a query parameter (city) via request.args.get().
3. Call OpenWeatherMap API:

url = f"http://api.openweathermap.org/data/2.5/weather?q=
{city}&appid=API_KEY&units=metric"
response = requests.get(url).json()

4. Extract and display:
response["main"]["temp"]
response["weather"][0]["description"]
response["sys"]["country"]
5. Handle errors (e.g., invalid city → show error message).
✅ Checkpoints for grading: API call works, correct fields displayed, error handling present.
